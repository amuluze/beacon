## 模块职责

`beacon` 的主要角色是：Server control plane: Web/API 接入、认证授权、Agent 生命周期、监控批次落库、目标选择和反向 tunnel client。

- 接收 Web/API 请求，负责认证授权、路由聚合、Agent 选择、Agent 列表查询和跨模块协调。
- 接收 Agent HTTP 监控上报批次，并把批次作为一致性单元写入 Server 本地监控表。
- 对容器、文件、系统等控制操作，通过 `common/rpc/tunnel` 按 Agent 标识调用目标 Agent。
- 相关源码包：`beacon/cmd/beacon`, `beacon/pkg/auth`, `beacon/pkg/auth/jwtauth`, `beacon/pkg/contextx`, `beacon/pkg/errors`, `beacon/pkg/fiberx`。
