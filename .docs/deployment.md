## 环境要求

- Go 运行环境需满足各模块 `go.mod` 声明的版本。
- 前端或 Node.js 包需安装依赖后运行对应 scripts。
- Docker/Compose 可作为本地或容器化启动入口。
- 运行前需要确认配置文件中的监听、数据库、Agent、凭据和路径语义。

## 快速启动

```bash
task website-server
cd beacon/web && npm install && npm run dev
cd website/web && npm install && npm run dev
docker compose -f deploy/docker-compose.yml up
docker compose -f website/compose.yml up
docker compose -f website/release/latest/compose.yaml up
```

## 部署方式

- 前端包通过 package scripts 运行本地开发、类型检查、lint 和构建。
- `deploy/docker-compose.yml` 提供 Docker Compose 启动入口。
- `website/compose.yml` 提供 Docker Compose 启动入口。
- `website/release/latest/compose.yaml` 提供 Docker Compose 启动入口。
- `beacon/Dockerfile` 提供镜像构建上下文。
