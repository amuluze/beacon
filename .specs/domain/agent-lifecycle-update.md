# Agent 生命周期与自更新（Agent Lifecycle & Self-Update）

> 状态: draft
> 最后更新: 2026-06-26
> 反推来源: local workspace facts + 需求分析

## 设计意图

约束 Agent 的注册、版本上报、版本感知、更新推送和卸载行为，保证 Server 能感知已部署 Agent 的版本并主动触发更新，Agent 能安全地完成自更新和自卸载。

本文件描述长期有效的领域约束：只写必须满足什么、允许什么、禁止什么，以及如何验证；不写实现步骤、临时文件路径、局部命名方案、可替换技术选型或伪代码。

## 约束空间

### 不变量（始终成立的断言）

| 编号 | 不变量 | 验收方式 |
|------|--------|----------|
| IU001 | Agent 注册到 Server 时必须携带可解析的版本标识；Server 必须持久化该版本。 | 注册帧包含版本字段，Server 侧 Agent 记录的 Version 字段非空。 |
| IU002 | Agent 版本标识必须单调递增（语义化版本或构建序号），Server 依据此标识判断新旧。 | 版本比较逻辑对合法版本格式返回确定的序关系。 |
| IU003 | Agent 自更新过程必须保证原子性：要么新二进制替换成功并重启，要么旧二进制继续运行；禁止处于中间不可用状态。 | 更新中断（网络断开、进程被杀）后，Agent 进程必须仍可启动并正常服务。 |
| IU004 | 更新或卸载操作必须可审计：操作发起方、目标 Agent、操作结果必须可追溯。 | Server 侧有操作日志或审计记录。 |

### 规则（允许 / 禁止）

| 编号 | 规则 | 允许/禁止 | 验收方式 |
|------|------|-----------|----------|
| RU001 | Server 必须能通过反向 tunnel 主动向指定 Agent 发起更新指令。 | 允许 | Server 调用更新 RPC 后，目标 Agent 开始下载新版本。 |
| RU002 | Agent 必须验证下载的二进制的完整性（校验和或签名）；校验失败必须拒绝替换并保留旧版本。 | 允许 | 下载被篡改的二进制后，Agent 不替换并上报校验失败。 |
| RU003 | Agent 更新必须由 Server 主动触发或 Agent 自主轮询触发；禁止第三方未授权触发。 | 禁止 | 无有效认证的更新请求被拒绝。 |
| RU004 | 卸载操作必须清理二进制、配置、数据和系统服务注册；禁止卸载后残留可执行文件。 | 禁止 | 卸载后 `/usr/sbin/collia`、`/etc/collia/`、服务注册均不存在。 |
| RU005 | 更新期间 Agent 必须先完成当前进行中的 RPC 调用或安全中断后才能退出；禁止更新导致进行中调用无响应。 | 禁止 | 更新时进行中的 terminal session 收到断开通知或优雅关闭。 |
| RU006 | 同一 Agent 不允许并发执行多个更新操作。 | 禁止 | 第二个更新请求被拒绝或排队，不会并行替换二进制。 |

## 对外可观测行为

| 场景 | 输入 | 可观测结果 |
|------|------|------------|
| Agent 注册 | Agent 通过 tunnel 连接 Server | Server 记录 Agent 的 agent_id、version、os、arch，Agent 列表 API 返回版本信息。 |
| Server 检测 Agent 版本落后 | 管理员查看 Agent 列表 | 版本落后的 Agent 在列表中标记为可更新状态。 |
| Server 推送更新 | 管理员对指定 Agent 触发更新 | Agent 下载新二进制 → 校验 → 替换 → 重启 → 以新版本重新注册。 |
| Agent 自主检查更新 | Agent 定时轮询 | Agent 发现新版本后下载 → 校验 → 替换 → 重启。 |
| 更新失败 | 下载失败或校验失败 | Agent 保留旧版本继续运行，Server 记录更新失败状态。 |
| 远程卸载 Agent | 管理员对指定 Agent 触发卸载 | Agent 停止服务 → 清理文件 → 断开 tunnel 连接，Agent 列表不再显示该 Agent。 |
| 本地卸载 Agent | 运维执行 collia remove | 系统服务注册移除，二进制、配置、数据目录清理。 |

## 输入 / 输出约束

### 输入合法性

- 合法输入：Agent 版本字符串（语义化版本格式）、目标 Agent 标识、更新源地址、卸载确认标记。
- 非法输入：格式错误的版本字符串、不存在的 Agent 标识、无认证的更新请求。
- 处理语义：非法输入必须被拒绝并返回可定位错误。

### 输出约束

- Agent 列表 API 必须包含每个 Agent 的 version 字段。
- 更新操作必须返回明确的成功或失败状态，禁止返回模糊结果。
- 更新进度（下载中/校验中/替换中/重启中）必须可查询。

## 状态模型

| 状态 | 含义 |
|------|------|
| `registered` | Agent 已连接 Server 并完成注册，携带版本信息。 |
| `update-available` | Server 判定 Agent 版本低于当前分发版本。 |
| `updating` | Agent 正在执行更新（下载/校验/替换）。 |
| `update-failed` | Agent 更新失败，回退到旧版本继续运行。 |
| `uninstalling` | Agent 正在执行卸载。 |
| `unregistered` | Agent 已卸载或断开连接。 |

### 状态迁移

| 起始状态 | 事件 | 目标状态 | 允许/禁止 |
|----------|------|----------|-----------|
| `registered` | Server 检测版本落后 | `update-available` | 允许 |
| `update-available` | 更新操作被触发 | `updating` | 允许 |
| `updating` | 下载、校验、替换全部成功 | `registered`（新版本） | 允许 |
| `updating` | 下载失败或校验失败 | `update-failed` | 允许 |
| `update-failed` | 重试更新 | `updating` | 允许 |
| `update-failed` | 放弃更新 | `registered`（旧版本） | 允许 |
| `registered` | 卸载操作被触发 | `uninstalling` | 允许 |
| `uninstalling` | 卸载完成 | `unregistered` | 允许 |
| `registered` | Agent 断开连接 | `unregistered` | 允许 |

## 错误条件与失败语义

| 错误条件 | 可观测表现 | 是否可恢复 | 是否幂等 |
|----------|------------|------------|----------|
| 下载中断 | Agent 上报下载失败，保留旧版本 | 是，可重试 | 是 |
| 二进制校验失败 | Agent 上报校验失败，保留旧版本 | 是，重新下载后重试 | 是 |
| 替换阶段进程被杀 | 旧二进制仍存在，下次启动运行旧版本 | 是，可重新触发更新 | 是 |
| 重启后新版本启动失败 | 根据回退策略决定：保留旧版本或标记为失败 | 是 | 否（需人工干预） |
| 卸载部分失败 | 已删除文件无法恢复，残留注册可重试清理 | 部分 | 否 |
| 更新请求发给离线 Agent | Server 返回 Agent 不可达错误 | 是，Agent 上线后重试 | 是 |
| 并发更新请求 | 第二个请求被拒绝或排队 | 是 | 是 |

## 结果约束（非功能性）

| 维度 | 约束 | 验收方式 |
|------|------|----------|
| 原子性 | 更新替换必须原子：新二进制 rename 后旧进程退出，不存在中间不可用状态 | 断电/杀进程测试 |
| 可追溯性 | 每次更新/卸载操作必须有审计记录 | 日志/审计查询 |
| 向后兼容 | 新版 Server 必须能与旧版 Agent 共存；旧版 Agent 不支持更新 RPC 时，Server 不得中断其正常功能 | 版本混合部署测试 |
| 安全性 | 更新通道必须复用已有认证机制（JoinToken 或 Install Token）；禁止明文传输密钥 | 安全审计 |
| 时效性 | Agent 版本信息延迟不得超过一个心跳周期（15s） | 心跳后版本一致性检查 |
