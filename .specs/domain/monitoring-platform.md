# 监控探测平台（Monitoring Platform）

> 状态: stable
> 最后更新: generated-by-sdd-doc-update

## 设计意图

约束 Server-Agent 监控探测平台的采集、控制、告警、Agent 选择、数据新鲜度和操作可观测性，保证用户看到的主机与容器运行状态可信。

本文件描述长期有效的领域约束：只写必须满足什么、允许什么、禁止什么，以及如何验证；不写实现步骤、临时文件路径、局部命名方案、可替换技术选型或伪代码。若路径本身是稳定协议表面，必须说明其契约身份与验收方式。

## 约束空间

### 不变量（始终成立的断言）

| 编号 | 不变量 | 验收方式 |
|------|--------|----------|
| I001 | 每个需要访问 Agent 的请求必须能解析出唯一目标 Agent；无法解析时必须使用明确的默认 Agent 或返回可见错误。 | 建议用 API/RPC 集成测试覆盖显式 Agent、默认 Agent 和未知 Agent。 |
| I002 | 主机、容器和告警数据必须携带可判断新鲜度的时间或采集批次信息；过期数据禁止展示为实时状态。 | 建议用数据查询测试覆盖新鲜、过期和空数据。 |
| I003 | 容器运行时的启动、停止、重启、删除、镜像拉取和网络变更必须只作用于被选中的目标节点。 | 建议在多节点场景下验证操作目标与审计记录一致。 |
| I004 | 告警状态必须由可观测指标、阈值配置和恢复条件共同决定，禁止只由前端展示状态推导。 | 建议用告警规则单测覆盖触发、保持、恢复和缺失数据。 |

### 规则（允许 / 禁止）

| 编号 | 规则 | 允许/禁止 | 验收方式 |
|------|------|-----------|----------|
| R001 | Agent 未连接、RPC 超时或权限不足时，Server 必须返回可区分错误，禁止降级为成功空结果。 | 禁止 | 建议用 API 错误语义测试验证。 |
| R002 | 容器运行时破坏性操作必须要求明确目标和权限上下文，禁止跨节点复用上一次操作目标。 | 禁止 | 建议用容器操作集成测试验证。 |
| R003 | 指标采集失败可以保留上一批数据，但必须显式标记为陈旧或降级。 | 允许 | 建议用数据新鲜度测试验证。 |
| R004 | 告警通知必须可追踪到触发指标、目标资源和当前状态。 | 允许 | 建议用告警记录与通知测试验证。 |

## 对外可观测行为

| 场景 | 输入 | 可观测结果 |
|------|------|------------|
| 查看监控面板 | 用户选择节点、容器或主机视图 | 返回目标 Agent 的最新可用指标，陈旧数据带降级标识。 |
| 执行容器操作 | 用户提交容器或镜像操作 | 只影响目标节点上的目标资源，并返回成功、失败或权限错误。 |
| 告警触发 | 指标达到阈值且满足持续条件 | 生成可追踪告警状态和用户可见通知。 |
| Agent 不可达 | 请求目标 Agent 时连接失败或超时 | 用户看到明确不可达错误，系统不伪造成功响应。 |

## 输入 / 输出约束

### 输入合法性

- 合法输入：目标节点标识、资源标识、指标查询条件、容器运行时操作参数、告警阈值和用户权限上下文。
- 非法输入：缺失目标节点的破坏性操作、无法验证权限的请求、无法映射到目标资源的操作参数。
- 处理语义：非法输入必须被拒绝并返回可定位错误；可恢复失败必须保留重试路径。

### 输出约束

- 每条关键不变量或规则必须有稳定编号和验收方式。
- 输出内容必须能被需求、实现计划或测试证据引用。
- 输出不得降低已有 Domain Spec 的约束强度。

## 状态模型

| 状态 | 含义 |
|------|------|
| `agent-disconnected` | Agent 不可达，Server 不能获取实时数据或执行控制操作。 |
| `agent-connected` | Agent 可达，可以接受查询或控制请求。 |
| `collecting` | Agent 正在采集主机、容器或系统指标。 |
| `degraded` | 最近一次采集或操作失败，系统只能展示陈旧数据或部分能力。 |
| `alarm-firing` | 指标满足告警触发条件且尚未恢复。 |
| `alarm-resolved` | 告警恢复条件已满足。 |

### 状态迁移

| 起始状态 | 事件 | 目标状态 | 允许/禁止 |
|----------|------|----------|-----------|
| `agent-disconnected` | Agent 握手或健康检查成功 | `agent-connected` | 允许 |
| `agent-connected` | 采集任务开始 | `collecting` | 允许 |
| `collecting` | 采集失败或数据过期 | `degraded` | 允许 |
| `degraded` | 新采集批次成功 | `agent-connected` | 允许 |
| `alarm-firing` | 恢复条件满足 | `alarm-resolved` | 允许 |
| `agent-disconnected` | 执行容器运行时破坏性操作 | `N/A（请求被拒绝）` | 禁止 |

## 错误条件与失败语义

| 错误条件 | 可观测表现 | 是否可恢复 | 是否幂等 |
|----------|------------|------------|----------|
| Agent 不可达 | API/RPC 返回目标不可达或超时 | 是，Agent 恢复后重试 | 是 |
| 指标过期 | 页面显示降级或陈旧状态 | 是，下一次采集成功后恢复 | 是 |
| 容器运行时操作失败 | 用户看到具体失败原因，目标资源状态不被伪造 | 视操作而定 | 视操作而定 |
| 告警通知失败 | 告警状态仍保留，通知通道记录失败 | 是，通知可重试 | 是 |

## 结果约束（非功能性）

| 维度 | 约束 | 验收方式 |
|------|------|----------|
| 可追踪性 | 约束 ID 必须可被需求、实现计划和测试证据回挂 | 引用检查 |
| 确定性 | 相同输入状态必须产生相同领域判断 | 重复执行同一验收场景 |
| 一致性 | 领域状态、可观测行为和错误语义必须互相匹配 | 评审与回归测试 |
| 安全性 | 不记录密钥、Token 或不可公开配置值 | 文档审计 |
