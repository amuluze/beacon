# Task: 监控图表时间窗口联动

> 依赖: Domain Spec [.specs/domain/monitoring-platform.md](../domain/monitoring-platform.md)
> 状态: done
> 优先级: P0 — 监控数据可信度

## 设计意图

让“时间密度”明确表示监控图表展示的最近时间窗口。用户选择 5 分钟、10 分钟、30 分钟等选项后，查询边界和图表横轴必须同时切换到对应的 `当前时间 - 窗口时长 → 当前时间`。

## 可验证行为约束

### 必须满足

| 编号 | 约束 | 验收方式 |
|------|------|----------|
| MTW-01 | 主机与容器查询必须同时应用 `start_time` 和 `end_time`，禁止返回窗口外样本 | Repository 测试覆盖窗口前、窗口内、窗口后数据 |
| MTW-02 | 图表横轴必须固定为本次请求的完整时间窗口，不能按返回样本的首尾时间自然缩放 | 组件测试断言 ECharts `xAxis.min/max` 等于请求边界 |
| MTW-03 | 每个序列数据点必须保留真实采集 timestamp | 组件测试断言 series 数据为 `[timestamp_ms, value]` |
| MTW-04 | 主机和容器监控必须支持近 5、10、30 分钟窗口 | 组件测试覆盖 5 分钟选项及窗口切换 |
| MTW-05 | 空窗口仍必须保留所选横轴范围，同时清空旧数据 | 组件测试覆盖空响应后的范围与 series |

### 禁止发生

| 编号 | 约束 | 验收方式 |
|------|------|----------|
| MTW-06 | 禁止把 timestamp 预先格式化为分类标签后丢弃原始时间 | 图表配置和 series 结构检查 |
| MTW-07 | 禁止旧请求响应覆盖最新 Agent 或时间窗口 | 现有竞态回归测试继续通过 |

## 错误语义

| 条件 | 结果 | 幂等 |
|------|------|------|
| 所选窗口无采集数据 | 展示完整空时间窗口，不保留上一窗口曲线 | 是 |
| 数据库存在窗口结束时间之后的数据 | 查询结果排除越界数据 | 是 |
