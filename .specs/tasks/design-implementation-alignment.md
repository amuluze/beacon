# Task: Beacon 设计稿实现对齐

> 依赖: Domain Spec [.specs/domain/monitoring-platform.md](../domain/monitoring-platform.md)
> 设计来源: [`.pen/beacon.pen`](../../.pen/beacon.pen)、[`.pen/website.pen`](../../.pen/website.pen)
> 状态: approved
> 优先级: P0 — 主要工作区可达性与官网设计一致性

## 设计意图

将当前后台与官网实现对齐已批准的 Pencil 设计稿，修复新版一级导航与旧多级路由并存造成的不可达页面，恢复设置页审计能力，并补齐容器编辑、重启策略和官网缺失页面。

## 可验证行为约束

### 必须满足

| 编号 | 约束 | 验收方式 |
|------|------|----------|
| DAI-01 | 后台一级导航的 Monitor、Container、Settings 必须分别呈现设计稿中对应的完整工作区，旧深链仍可访问 | 从顶栏进入三个工作区，主机/容器监控、容器/镜像/网络管理、系统/告警/镜像仓库/审计均有可见入口 |
| DAI-02 | Settings 工作区必须恢复按当前 Agent 与审计类型查询的审计日志，列表与 total 必须使用相同过滤条件 | 选择 Agent 后，审计请求显式携带 Agent 标识；分页 total 与当前筛选结果一致 |
| DAI-03 | 容器创建和编辑必须支持 `no`、`always`、`unless-stopped`、`on-failure` 重启策略 | 创建/编辑请求通过 Server 与 tunnel 传递到目标 Agent；非法策略返回可见错误 |
| DAI-04 | 容器编辑必须使用显式目标 Agent，并在替换失败时尽量恢复原容器名称与原运行状态 | 单元测试覆盖成功、缺失/非法 Agent、显式 Agent 透传、创建新容器失败回滚、重启策略更新失败和原运行状态恢复 |
| DAI-05 | 添加/编辑容器使用 480px Drawer；拉取/导入镜像、新建网络、告警阈值和邮件编辑使用 480px Modal；镜像仓库等长表单允许 540px Drawer；全部保留 loading/error/success 状态 | 前端组件属性、失败反馈与相关交互测试/类型检查通过 |
| DAI-06 | 官网必须实现新版 Header、Footer、首页、关于、更新日志和使用手册导航结构 | Header 展示首页/使用手册/GitHub，Footer 展示团队故事/使用手册/微信公众号/GitHub；Nuxt 路由与关键区块对应设计稿 |
| DAI-07 | 官网在 375px 宽度下必须使用精简响应式布局，不出现水平溢出 | 构建通过，移动断点样式覆盖导航、Hero、功能区和页脚 |

### 禁止发生

| 编号 | 约束 | 验收方式 |
|------|------|----------|
| DAI-08 | 禁止后台一级菜单只跳到默认子路由而隐藏其余子页面 | 路由与导航组件测试/静态断言 |
| DAI-09 | 禁止取消邮件编辑时提前修改父级设置，禁止 API 失败后提示成功 | 组件逻辑使用本地副本；失败分支不触发 success |
| DAI-10 | 禁止官网继续使用紫蓝渐变、渐变文本和旧轮播图作为主视觉 | 源码扫描与页面结构检查 |
| DAI-11 | 禁止官网统计接口失败导致首页主体白屏或未捕获异常 | 统计失败时显示 `--` 安全占位，Hero 与其余区块正常渲染 |

## 输入 / 输出

- **输入**：当前 Agent、容器配置、官网路由访问、设计稿定义的交互操作。
- **输出**：可达的聚合工作区、可追踪容器编辑结果、与设计稿一致的官网页面结构。

## 错误语义

| 条件 | 可观测结果 | 是否可恢复 |
|------|------------|------------|
| Agent 未选择或离线 | 控制请求返回明确 Agent 错误，不操作其他节点 | 是 |
| 容器替换失败 | 返回具体失败原因；能恢复时恢复原名称与运行状态 | 视 Docker 状态而定 |
| 审计查询失败 | 审计区显示错误/空态，不伪造成功数据 | 是 |
| 官网统计接口不可用 | 首页主体仍可展示，统计值使用安全占位 | 是 |

## 视觉验收

- 桌面端以 1440px 视口分别对照后台 Monitor / Container / Settings 与官网 Landing / About / Changelog / Docs。
- 移动端以 375px 视口对照 `Landing Mobile`，检查导航、Hero、功能区与 Footer 无水平溢出。
- 若执行环境没有可用浏览器，必须把视觉验收标记为未完成并说明阻塞，不得以源码检查冒充截图验收。
